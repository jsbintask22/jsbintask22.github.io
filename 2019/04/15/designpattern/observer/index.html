从未这么明白的设计模式（二）：观察者模式

recommend
  • java
  • 设计模式


本文原创地址，我的博客：https://jsbintask.cn/2019/04/15/designpattern/observer/(食用效果最佳)，转载请注明出处!

前言

观察者模式定义了对象间的一种一对多依赖关系，当一个对象状态发生改变时，观察者们都可以做出相应的更新，使得系统更易于扩展！
代码地址：https://github.com/jsbintask22/design-pattern-learning

案例

  1. 小丽长得很漂亮，”天生丽质难自弃”, 是一个不折不扣的”女神”。
  2. 小丽身边有很多”备胎“，他们通过各种方式添加了小丽的微信，“小豪，小吴”都是其中之一。
  3. 小丽总是会在朋友圈发布自己的各种生活状态。
  4. ”备胎们“总是及时并且积极地和女神互动！
  5. 小丽发现”备胎“小豪不爱互动了，于是删除了“备胎”小豪的微信。
  6. 小豪发现自己看不了女神动态了。最终死心！
  7. 小丽认识了新“备胎”小李，于是小李也添加了女神微信。
  8. 小丽发布自己的朋友圈动态，小李也开始了互动！
    上面这个过程我们可以抽象出来两个主题，女神小丽,备胎小豪，小吴，小李，我们用代码模拟这个追女神的过程。
